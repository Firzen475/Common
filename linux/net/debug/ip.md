## Основные возможности

- Управление сетевыми интерфейсами (включение/выключение, настройка параметров)
- Конфигурация IP-адресов и маршрутов
- Управление ARP- и NDISC-кешем
- Настройка политик маршрутизации
- Работа с VLAN, VPN, мостами и туннелями

## Основные подкоманды

| Подкоманда                          | Описание                            | Примеры                                                                                    |                                                      
|-------------------------------------|-------------------------------------|--------------------------------------------------------------------------------------------| 
| `ip addr` (`show,add,del,flush`)    | Работа с IP-адресами                | `ip addr add 192.168.1.100/24 dev eth0`<br/>`ip addr flush eth0`                           |
| `ip link` (`show,set[up,down,mtu]`) | Управление сетевыми интерфейсами    | `ip link set eth0 up`<br/>`ip link set eth0 mtu 1500`                                      |
| `ip route` (`show,add,del,get`)     | Управление таблицей маршрутизации   | `ip route add 192.168.1.0/24 via 192.168.0.1`<br/>`ip route get 8.8.8.8`                   |
| `ip neigh` (`show,add,del,flush`)   | Управление ARP-таблицей             | `ip neigh add 192.168.1.1 lladdr 00:11:22:33:44:55 dev eth0`<br/>`ip neigh flush dev eth0` |
| `ip rule`                           | Управление политиками маршрутизации |                                                                                            |
| `ip maddr`                          | Управление multicast-адресами       |                                                                                            |
| `ip tunnel`                         | Работа с туннелями                  |                                                                                            |
| `ip monitor`                        | Мониторинг сетевых событий          |                                                                                            |

## Состояния сетевого интерфейса
| Состояние     | Описание                                                               |
|---------------|------------------------------------------------------------------------|
| `UP`          | Интерфейс включён (администрирование разрешено).                       |
| `DOWN`        | Интерфейс выключен (администрирование запрещено).                      |
| `UNKNOWN`     | Неизвестное состояние (кабель подключён, но линка нет).                |
| `LOWER_UP`    | Физический уровень активен (есть линк, кабель подключён).              |
| `LOWER_DOWN`  | Физический уровень неактивен (нет линка, кабель отключён).             |
| `NO-CARRIER`  | Нет несущей (кабель есть, но сигнала нет).                             |
| `DORMANT`     | Спящий режим (ожидает активации, например, PPPoE).                     |
| `LOOPBACK`    | Интерфейс является loopback-устройством (`lo`).                        |
| `PROMISC`     | Режим promiscuous (принимает все пакеты, даже не свои).                |
| `ALLMULTI`    | Принимает все multicast-пакеты.                                        |
| `MULTICAST`   | Поддерживает multicast.                                                |
| `BROADCAST`   | Поддерживает широковещательные пакеты (broadcast).                     |
| `POINTOPOINT` | Режим "точка-точка" (например, PPP, VPN).                              |
| `SLAVE`       | Подчинённый интерфейс (в bonding/teaming).                             |
| `MASTER`      | Ведущий интерфейс (в bonding/bridge).                                  |
| `DYNAMIC`     | Поддерживает динамическое изменение MAC-адреса.                        |
| `AUTOMEDIA`   | Автоматическое определение типа среды (Ethernet, 10/100/1000 и т. д.). |
| `ECHO`        | Может отправлять эхо-ответы (редко используется).                      |
