# Сравнение SELinux и AppArmor

| Критерий                    | SELinux                                                                  | AppArmor                                                          |
|-----------------------------|--------------------------------------------------------------------------|-------------------------------------------------------------------|
| **Тип контроля доступа**    | Mandatory Access Control (MAC) на основе меток (контекстов безопасности) | Path-based MAC (профили, привязанные к путям файлов)              |
| **Сложность настройки**     | Высокая (требует глубокого понимания)                                    | Средняя (более простые профили)                                   |
| **Гибкость**                | Очень высокая (поддержка MLS/MCS)                                        | Ограниченная (нет поддержки многоуровневой безопасности)          |
| **Логирование**             | `audit.log` (интеграция с auditd)                                        | `syslog`/`kern.log`                                               |
| **Управление**              | `semanage`, `setsebool`, `chcon`, `restorecon`                           | `aa-genprof`, `aa-enforce`, `aa-complain`                         |
| **Пример проверки статуса** | `sestatus`                                                               | `aa-status`                                                       |
| **Отключение**              | `setenforce 0` (временно) или редактирование `/etc/selinux/config`       | `systemctl stop apparmor` + `systemctl disable apparmor`          |


# AppArmor
* Создаётся профиль или используется default
  * Доступ к файлам\каталогам по маске rwx.
  * Разрешения на запуск других программ.
  * Сетевые ограничения (deny network inet6).
* Автогенерация правил
* Работа на основе путей в правилах.
* После добавления правила анализируются ВСЕ системные вызовы в соответствии с ним.
* Режимы работы enforce(блокировка)\complain(логирование)
