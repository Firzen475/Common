# CFS (Completely Fair Scheduler)
* Каждая задача (процесс/поток) получает виртуальное время выполнения (vruntime).
* Чем дольше задача использует CPU, тем больше растёт её vruntime.
* Задача с наименьшим vruntime получает CPU (очередь строится по принципу красно-чёрного дерева).
* Приоритета задачи (nice-значение от -20 до 19).
* Чем меньше nice - тем меньше увеличивается vruntime_new
  * Формула: `vruntime += (runtime * 1024) / (weight = 1024 / (1.25)^nice)`
* При изменении nice: `vruntime_new = vruntime_old * (weight_old / weight_new)`
