
# Создать копию диска
```bash 
sudo umount /dev/sdX
dd if=/dev/sdX of=/backup.img
```
* Потом смонтировать в режиме RO

# Методы восстановления
| Тип восстановления                      | Описание                                                                  | Используемые утилиты          | Пример команды                                                          |
|-----------------------------------------|---------------------------------------------------------------------------|-------------------------------|-------------------------------------------------------------------------|
| **Проверка/исправление ФС**             | Восстановление структуры ФС (суперблок, inode, битовые карты) после сбоя. | `fsck` (`e2fsck`)             | `sudo fsck -y /dev/sdX`                                                 |
| **Восстановление суперблока**           | Использование резервных копий суперблока при повреждении основного.       | `fsck`, `mke2fs` (для поиска) | `sudo fsck -b 32768 /dev/sdX` (где `32768` — резервный суперблок)       |
| **Восстановление удалённых файлов**     | Поиск и восстановление файлов по метаданным inode.                        | `extundelete`, `testdisk`     | `sudo extundelete /dev/sdX --restore-all`                               |
| **Ручное восстановление через debugfs** | Точечное восстановление файлов по известным inode.                        | `debugfs`                     | ```sudo debugfs /dev/sdX```<br>`debugfs: dump <123456> /recovered_file` |
| **Восстановление структуры каталогов**  | Восстановление дерева каталогов и имён файлов.                            | `ext4magic`, `scalpel`        | `sudo ext4magic /dev/sdX -r -f /home/user/Documents -d /recovery`       |
| **Восстановление после форматирования** | Поиск данных после перезаписи раздела (без гарантии сохранения имён).     | `testdisk`, `photorec`        | ```sudo testdisk /dev/sdX``` (интерактивный режим)                      |
