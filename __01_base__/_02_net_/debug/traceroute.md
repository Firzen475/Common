### `traceroute`: диагностика сетевых маршрутов

`traceroute` — это утилита для отслеживания маршрута пакетов в сети и диагностики задержек между узлами.

---

## Как работает `traceroute`?
- Отправляет последовательность UDP-пакетов с увеличивающимся TTL (Time To Live).
- Каждый маршрутизатор уменьшает TTL на 1. Когда TTL становится 0, узел отвечает ICMP-сообщением об ошибке.
- `traceroute` фиксирует IP-адрес узла и измеряет задержку.

---

## Основные ключи `traceroute`

| Ключ         | Описание                                         |
|--------------|--------------------------------------------------|
| `-n`         | Вывод только IP-адресов без DNS-имен             |
| `-m <число>` | Ограничение максимального количества хопов (TTL) |
| `-w <сек>`   | Таймаут ожидания ответа от узла                  |
| `-q <число>` | Количество запросов на один хоп (по умолчанию 3) |
| `-I`         | Использовать ICMP вместо UDP                     |
| `-T`         | Использовать TCP вместо UDP                      |

---

## Примеры использования

##
```sh
# 1. Обычный запуск: Выведет маршрут к серверу `example.com` с указанием всех промежуточных узлов.
traceroute example.com
# 2. Вывод только IP-адресов:
traceroute -n example.com
# 3. Уменьшение количества отправляемых пакетов: Отправляет по 1 пакету на каждый хоп (по умолчанию 3).
traceroute -q 1 example.com
# 4. Использование ICMP (как `tracert` в Windows):
traceroute -I example.com
# 5. Ограничение числа хопов: Ограничивает трассировку 10 хопами.
traceroute -m 10 example.com
```
